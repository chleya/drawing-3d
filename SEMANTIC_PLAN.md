## 三、分阶段实施计划

### Phase 1: 基础能力 (1-2周)
**目标：** 让系统"能问答"

| 任务 | 内容 | 交付物 |
|------|------|--------|
| 1.1 知识图谱增强 | 添加更多工程结构模板 (路面/桥梁/涵洞) | 扩展的 layer_templates |
| 1.2 问答系统升级 | 接入LLM API (OpenAI/MiniMax) | blueprint_qa_v2.py |
| 1.3 Web集成 | 在Web界面添加"智能问答"入口 | web_server.py 更新 |

### Phase 2: 空间能力 (2-3周)
**目标：** 让系统"能定位"

| 任务 | 内容 | 交付物 |
|------|------|--------|
| 2.1 坐标引擎 | 支持平曲线、竖曲线计算 | coordinate_engine.py |
| 2.2 照片关联 | 照片GPS自动归档功能 | photo_mapper.py |
| 2.3 三维集成 | Cesium/Three.js 场景集成 | 3d_view 模块 |

### Phase 3: 自动化 (3-4周)
**目标：** 让系统"能学习"

| 任务 | 内容 | 交付物 |
|------|------|--------|
| 3.1 图纸解析 | CAD/DXF 文件自动解析 | cad_parser.py |
| 3.2 数据融合 | 多源数据 (设计/施工/检测) 融合 | data_fusion.py |
| 3.3 预测分析 | 进度/质量预测 | prediction_engine.py |

## 四、技术选型

### 当前技术栈
```
后端:     Python (Flask/FastAPI)
数据库:   JSON文件 (当前) → PostgreSQL (计划)
图数据库: 内存模拟 (当前) → Neo4j (计划)
AI:      模板匹配 (当前) → LLM API (计划)
三维:    基础3D (当前) → CesiumJS (计划)
```

### 推荐技术升级
| 层级 | 推荐方案 | 理由 |
|------|----------|------|
| 图数据库 | Neo4j Desktop | 免费、易用、支持空间查询 |
| LLM | MiniMax API | 中文优化、性价比高 |
| GIS | CesiumJS | 开源、支持大场景 |
| CAD解析 | ezdxf | Python原生、免费 |
